## Findings considered and rejected

- **".env committed with live secrets"** (subagent report): false — `git
  ls-files .env` and `git log --all -- .env` are both empty; the file is
  local-only and gitignored. Not re-auditable.
- **Session cookie `secure` flag conditional on NODE_ENV**: standard
  framework convention, not a finding.
- **Xbox callback account-linking CSRF/IDOR**: the callback validates a
  per-browser state cookie (`src/app/api/auth/xbox/callback/route.ts:24`);
  the reported attack requires the victim to hold the attacker's state
  cookie, which they can't.
- **Steam OpenID verification weakness**: `verifySteamOpenIdCallback` posts
  the assertion to the hardcoded Steam endpoint (`check_authentication`,
  `src/lib/steam.ts:90-104`); Steam enforces nonce single-use. Adequate.
- **scrypt cost parameters not explicit** (`src/lib/password-auth.ts`):
  Node's defaults are exactly the recommended N=16384,r=8,p=1. Cosmetic.
- **JWT algorithm confusion in `jwtVerify`**: jose derives acceptable
  algorithms from key type and always rejects `none`; symmetric-key HS-family
  confusion is not exploitable here.
- **Server-action CSRF**: Next.js server actions enforce origin checks;
  by-design.
- **Unused dependencies**: checked (`@ai-sdk/react`, `react-markdown`,
  `psn-api`) — all in use.
- **Game-detail enrichment "unconditional sequential calls"**: overreported —
  `enrichMissingGameDetailData` (`src/lib/catalog.ts:1199-1254`) already
  guards each call on missing data; residual sequentiality only matters when
  several fields are missing at once. Not worth doing now.
- **Prisma index additions**: existing `@@index` coverage matches observed
  query patterns; nothing to add.
