## Vetted but not planned (confirmed findings, available for future plans)

- **Provider sync orchestration triplicated** in `src/lib/catalog.ts`
  (Steam, PlayStation, Xbox): near-identical resolve/upsert loops that have
  already drifted. `ProviderAccountAdapter` exists in
  `src/lib/providers/contracts.ts` but only models Steam. Effort L; do only
  after the current catalog/data-loading plans settle.
- **`src/lib/i18n.ts` is imported by many client and server files**: the file is
  about 104 KB and contains both locales. It likely deserves a bundle-analysis
  spike before a split. Effort M, confidence MED.
- **Sync loop latency architecture**: even after plan 005, first-time syncs do
  sequential per-game external calls and several DB round-trips per title.
  Bounded concurrency plus batched prefetch is the follow-up track. Effort L.
- **OAuth callback redirects embed raw internal error messages** in callback
  query parameters. Low-impact information disclosure; replace with generic
  user-facing copy plus server-side logging. Effort S.
- **CSV import marks jobs COMPLETED regardless of per-row failures**:
  `ImportJob.summary` carries counts, but status semantics can mislead. Effort
  S; product decision needed before changing audit semantics.
- **Moderate PostCSS advisory remains through Next's nested dependency**:
  `npm audit --omit=dev` reports moderate advisories only. Track through
  framework updates unless severity changes.

## Direction findings (maintainer options, evidence-based - not ranked against bugs)

- **Vestigial YouTube OAuth identity path**: `User.youtubeSubject` and
  `googleSubject` both store the same Google OIDC subject, and
  `src/app/api/auth/youtube` is a second Google login path. Consolidating on
  `googleSubject` simplifies future auth changes. Coarse effort S.
- **Photo import is built but buried**: full AI-vision import pipeline exists
  and is admin-gated, but it is not as prominent as the provider/CSV source
  flows. Coarse effort M. Respect the calm-UI constraint in `AGENTS.md`.
- **Finished-game detection is sync-triggered only**: story-completion
  detection runs on sync, so credits-rolled games stay unmarked until manual
  sync. A scheduled or login-triggered recheck would close the loop. Coarse
  effort M.

## Findings considered and rejected

- **".env committed with live secrets"**: false. `git ls-files .env` and
  `git log --all -- .env` were empty; the file is local-only and gitignored.
- **Session cookie `secure` flag conditional on NODE_ENV**: standard framework
  convention, not a finding.
- **Xbox callback account-linking CSRF/IDOR**: the callback validates a
  per-browser state cookie; the reported attack requires the victim to hold the
  attacker's state cookie.
- **Steam OpenID verification weakness**: `verifySteamOpenIdCallback` posts the
  assertion to Steam's `check_authentication` endpoint; adequate.
- **scrypt cost parameters not explicit**: Node's defaults match the expected
  N=16384,r=8,p=1 parameters. Cosmetic.
- **JWT algorithm confusion in `jwtVerify`**: jose rejects `none` and derives
  acceptable algorithms from key type; not exploitable here.
- **Server-action CSRF**: Next.js server actions enforce origin checks;
  by-design.
- **Unused dependencies**: checked `@ai-sdk/react`, `react-markdown`, and
  `psn-api`; all are in use.
- **Game-detail enrichment unconditional sequential calls**: overreported.
  `enrichMissingGameDetailData` already guards calls on missing data.
- **Generic Prisma index additions**: existing index coverage matched observed
  query patterns during the audit.
